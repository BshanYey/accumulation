面向对象设计原则
1、单一职责原则（SRP）
    一个类只负责一个功能领域的相应职责。（高内聚，低耦合）
2、开闭原则（OCP）
    一个软件的实体应当对修改关闭，对扩展开放
3、依赖倒转原则（DIP）
    要针对接口编程，不要针对实现编程
4、迪米特法则（LOD）
    一个类指引入相关的类，无关的类不要引入
5、接口隔离原则（ISP）
    接口不要过于臃肿，通过接口拆分降低耦合
6、里氏代换原则（DP）
    任何基类可以出现的地方子类一定可以出现（子类可以拓展父类的功能，但不能改变父类的功能）
